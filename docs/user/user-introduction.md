# AutoMedia 用户介绍

## 一句话概括

**AutoMedia 是一个自动化内容生产系统。你给一个选题，它自己完成写稿、核验、配音、做视频、生成字幕、分发到多个平台的全流程。**

它不是又一个"AI写作工具"。它是一条完整的内容生产线 —— 从想法到发布，中间 33 道质量关卡全部自动化，最后成品直接出现在你的公众号、知乎、飞书群里。

---

## 它解决了什么实际问题

### 你现在的痛苦

| 问题 | 你现在的做法 | 代价 |
|------|-------------|------|
| 每天要写多篇内容 | 人工写，或每篇都去 ChatGPT 复制粘贴 | 时间全耗在重复劳动上 |
| 内容质量不稳定 | 写完就发，没人审核 | 出错、品牌调性不一致、被平台判违规 |
| 多平台发布太麻烦 | 一篇内容手动适配微信、知乎、小红书 | 每个平台格式不同，工作量翻倍 |
| 视频内容制作成本高 | 外包或自己学剪辑 | 一篇文章到一条视频至少半天 |
| 品牌一致性难保证 | 不同人写的东西风格差异大 | 品牌形象模糊，读者认不出你 |
| 海外内容本地化 | 找翻译，反复沟通 | 周期长，成本高，术语不统一 |

### AutoMedia 的解决方式

- **你只需给一个选题**（比如"AI视频工具对比"），系统自动完成：选题判断 → 写稿 → 事实核查 → 人性化润色 → 品牌合规检查 → 配音 → 字幕 → 视频合成 → 多平台发布
- **33 道质量关卡自动执行**（含 G0-G6 文字门、V0-V7 视频门、D1-D7 分发门、P1-P4 复用门）：事实有没有错、是不是AI味太重、品牌名有没有写对、CTA有没有漏、视频字幕清不清晰、翻译质量过不过关 —— 全部自动检查
- **一次生产，多处分发**：同一篇内容自动适配微信公众号、知乎、飞书通知等平台
- **不是黑盒**：每一步的结果都可以查阅、修改、人工审核后再放行

## 它具备哪些能力

### 内容生产全流程自动化

```
你的输入                           AutoMedia 处理链                           最终输出
───────                          ─────────────                              ────────
一个选题 (topic)       →   选题筛选 → AI写稿 → 事实核查                         → 公众号文章
一个品牌 (brand)       →   → 人性化润色 → 品牌CTA检查                          → 知乎回答
一个模式 (mode)        →   → HTML合规 → 配音 → 字幕生成                        → 视频文件
                         → 视觉质检 → 语义覆盖检查 → 最终核验                    → 飞书通知
                         → 归档 → 多平台发布
```

### 内容生成与质量保障

- **AI 写稿**：基于你的品牌调性、语气风格、CTA原则自动生成文章
- **事实核查**（G0）：自动验证文中的数字、引文、实体名称、时间线是否与源数据一致
- **AI味去除**（G1）：自动检测并去除 9 类 AI 写作痕迹（套路化开头、空洞衔接词、过度学术用语、绝对断言等）
- **文案审查**（G2）：检查清晰度、语调一致性、是否有论据支撑、是否具体不空泛
- **品牌合规**（G3）：品牌名有没有写对、CTA 有没有漏、违禁词有没有出现
- **微信合规**（G4）：标题长度、摘要长度、封面图、标签数量、正文配图数量、敏感词 —— 全部自动检查
- **HTML 完整性**（G5）：标签是否配对、有没有 Markdown 残留

### 视频与音频制作

- **自动配音**：利用 edge-tts 将文字转为语音（支持多语言、多音色）
- **字幕生成**：Whisper 语音识别 → 自动生成 SRT 字幕文件
- **字幕质检**（V6）：像素级检查字幕亮度、对比度、可见性 —— 确保最终视频里字幕清晰可读
- **视频合成**：自动生成封面图（4种比例，由 ComfyUI 生成或 PIL 降级为占位图）、正文配图、合成视频
- **语音一致性**（V4）：检查所有片段的语音 ID、语速是否一致
- **转录完整性**（V2）：确保完整音频被转录，不允许抽样

### 多语言与本地化

- **文档内容提取**（OPP）：从 .md、.txt 等格式中提取结构化内容（.docx/.pptx/.pdf 需安装 omni 扩展）
- **内容翻译**（OL）：基于 shield → LLM翻译 → 修复 → 解除防护 的管线进行多语言翻译
- **翻译质量检查**（L4）：自动验证翻译有无乱码、语言标签是否正确、内容非空
- **格式转换**（ORF）：将翻译后的内容转换回目标格式

### 多平台发布

- **微信公众号**：自动获取 access_token → 创建草稿 → 提交发布
- **知乎**：自动创建知乎草稿
- **飞书**：自动发送交互式卡片通知到飞书群
- **账户管理**：支持 OAuth2、Cookie、API Key 多种认证方式，凭据 AES-256-GCM 加密存储

### 内容资产管理

- **知识库**：每个品牌拥有独立的 SQLite + 向量搜索知识库，历史内容可检索、可复用
- **智能搜索**：关键词 + 语义相似度混合搜索，支持 3 层广度优先扩展发现相关内容
- **内容入库**：生产完成的内容可入库、去重、打标签（通过 automedia asset ingest 手动触发）

## 它能用在什么场景

### 场景一：自媒体内容矩阵运营

> **你运营 3 个公众号 + 知乎账号，每天需要产出 5 篇内容。**

传统方式：5 篇文章 × 写稿 1 小时 × 3 个平台 = 每天至少 6–8 小时。

AutoMedia：给 5 个选题 → 自动写稿 → 自动检查 → 自动发布。你只需要审核确认，每天 30 分钟。

### 场景二：品牌内容团队

> **你是一个品牌的运营负责人，要保证所有对外内容品牌调性一致。**

AutoMedia 的 G3（品牌CTA门）自动检查每篇文章是否：
- 正确使用了品牌名称
- 包含了正确的 CTA
- 没有出现违禁词
- 品牌身份定位一致

不再需要人工逐篇审核。

### 场景三：出海内容本地化

> **你要把中文内容翻译成英语、日语、韩语，发布到海外平台。**

OPP 提取源文件 → OL 翻译管线（含术语库、风格指南、质量闭环）→ L4 翻译质量检查 → ORF 格式转换 → 多语言内容入库。全自动化。

### 场景四：AI Agent 驱动的自动化工作流

> **你使用 Claude Code、OpenCode 或 Codex CLI 等 AI 编程助手，希望它们能直接操作你的内容生产流水线。**

AutoMedia 提供 MCP 服务器（52 个工具），AI Agent 可以直接调用：
- `run_pipeline` → 启动生产流水线
- `get_pipeline_progress` → 实时查看进度
- `publish_content` → 发布到平台
- `extract_brief` → 提取文档内容
- `localize_content` → 翻译内容

你只需要对 AI Agent 说"帮我把这篇文档提取出来，翻译成英文，然后生成视频发布到公众号"，Agent 就能调用 AutoMedia 完成。

### 场景五：定时化内容生产

> **你希望每天早上 8 点自动抓取热点话题，评分排序，8 点半自动开始生产。**

配置好 crond + AutoMedia 的定时任务，每天自动：
1. 08:00 — 收集热点话题到选题池
2. 08:05 — 对话题评分排序
3. 08:30 — 检查待发布内容
4. 09:30 — 系统健康检查

你只需要在早上打开电脑看一眼结果。

## 它不做什么

诚实地说清楚边界：

| 不做 | 原因 |
|------|------|
| 不做用户界面/网页 | 这是命令行 + MCP 工具，不是 SaaS 后台。没有登录页面、没有 Dashboard。 |
| 不做内置调度器 | 定时任务依赖系统 crond 触发，AutoMedia 本身不常驻后台。 |
| 不做小红书自动发布 | 小红书没有公开 API，无法自动发布。 |
| 不做原创图片/视频生成 | 封面图依赖 ComfyUI（需自行部署）或 PIL 生成占位图，不是 Midjourney。 |
| 不替代人工审核 | 33 道质量关卡是自动检查，不是自动通过。你可以配置 HITL 模式让人工介入关键节点。 |

---

## 一句话总结

**AutoMedia 是你内容生产团队的"自动驾驶系统"** —— 它不是帮你写一篇更好的文章，而是把从选题到发布的全流程工业化、自动化、质量可控。你给它方向和选题，它完成剩下的所有体力活。
